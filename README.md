# 明日方舟基建计算器

## 项目概述

这是一个用于计算《明日方舟》游戏中基建系统效率的模拟器。通过配置文件输入干员部署情况，自动计算各设施的生产效率和特殊变量产出。

## 项目结构

```
building-calculator/
├── main.py                      # 主程序入口
├── example_config.json           # 配置示例文件
├── core/
│   ├── __init__.py
│   ├── base_classes.py          # 基础类（特殊变量存储器、心情系统、干员基类）
│   ├── operator_classes.py      # 所有干员类定义
│   ├── building_simulator.py    # 基建模拟器主程序
│   ├── facility_calculators.py  # 设施效率计算器
│   ├── station_calculator.py     # 制造站效率计算器
│   └── trade_station_calculator.py  # 贸易站效率计算器
└── data/                         # 数据处理脚本
```

## 核心概念

### 特殊变量系统

特殊变量是干员技能产生的全局资源，可被多个干员使用和转化：

| 变量名 | 说明 | 示例来源 |
|--------|------|----------|
| 感知信息 | 全局共享，来源多样 | 迷迭香、黑键、爱丽丝等 |
| 思维链环 | 感知信息转化 | 迷迭香(E2) |
| 无声共鸣 | 感知信息转化 | 黑键(E2) |
| 人间烟火 | 感知信息转化 | 夕、令、乌有 |
| 小节 | 车尔尼特殊变量 | 宿舍等级相关 |
| 梦境 | 爱丽丝特殊变量 | 宿舍等级相关 |
| 记忆碎片 | 絮雨特殊变量 | 办公室招募位 |

### 心情系统

- 心情上限：24点（满心情）
- 所有干员默认心情为满
- 夕、令等干员的技能效果会根据心情值变化

### 计算流程

```
1. 收集特殊变量 - 初始化所有干员的特殊变量列表
2. 计算特殊技能 - 遍历所有设施，计算干员技能产生的特殊变量
2.5 计算感知信息转化 - 后处理（如迷迭香、黑键的感知信息转化）
3. 计算效率 - 各设施计算基础效率+干员效率
4. 计算产出 - 最终产出结果
```

## 干员分类

### 宿舍干员

| 干员 | 精英等级 | 特殊变量 | 技能效果 |
|------|----------|----------|----------|
| 迷迭香 | E2 | 感知信息、思维链环 | 宿舍每1人+1感知信息；感知信息→思维链环 |
| 黑键 | E2 | 感知信息、无声共鸣 | 宿舍每1人+1感知信息；感知信息→无声共鸣 |
| 爱丽丝 | E2 | 梦境、感知信息 | 梦境=宿舍等级；梦境→感知信息 |
| 车尔尼 | E2 | 小节、感知信息 | 小节=宿舍等级；小节→感知信息 |
| 塑心 | E2 | 无声共鸣 | 宿舍每1人+1无声共鸣 |
| 乌有 | E0 | 人间烟火 | 宿舍每1人+1人间烟火 |

### 控制中枢干员

| 干员 | 精英等级 | 特殊变量 | 技能效果 |
|------|----------|----------|----------|
| 夕 | E2 | 感知信息/人间烟火 | 心情>12:感知信息+10；心情≤12:人间烟火+15 |
| 令 | E2 | 感知信息/人间烟火 | 心情>12:人间烟火+15；心情≤12:感知信息+10 |
| 重岳 | E2 | 人间烟火 | 每名岁干员+5人间烟火(最多5名) |

### 办公室干员

| 干员 | 精英等级 | 特殊变量 | 技能效果 |
|------|----------|----------|----------|
| 絮雨 | E2 | 记忆碎片、感知信息 | 每招募位(不含初始)+10记忆碎片；记忆碎片→感知信息 |

### 制造站/贸易站干员

提供赤金效率、作战记录效率、订单效率等属性加成。

## 关键设计

### 1. 感知信息转化后处理

迷迭香和黑键的感知信息转化在步骤2.5统一处理，确保所有感知信息累加后再进行转化：

```python
def 计算所有感知信息转化(self):
    """步骤2.5: 后处理感知信息转化"""
    for 设施 in self.所有设施:
        for op in 干员列表:
            干员实例 = 创建干员实例(...)
            if hasattr(干员实例, '计算感知信息转化'):
                干员实例.计算感知信息转化()
```

### 2. 特殊变量来源追踪

每个特殊变量的来源和数值都被记录：

```python
class 特殊变量存储器:
    def __init__(self):
        self.变量字典 = {}     # {"感知信息": 0, ...}
        self.来源记录 = {}     # {"感知信息": [("来源1", 值1), ...], ...}
```

### 3. 干员类工厂模式

通过注册表统一创建干员实例：

```python
OPERATOR_REGISTRY = {
    "迷迭香": 迷迭香,
    "黑键": 黑键,
    ...
}

def 创建干员实例(名称, 精英等级, 特殊变量, 配置数据):
    干员类 = OPERATOR_REGISTRY[名称]
    return 干员类(精英等级, 特殊变量, 配置数据)
```
