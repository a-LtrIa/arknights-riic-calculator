# 基建效率计算方法

> 本文档描述明日方舟基建模拟器的完整计算模型。核心结论是：**计算不是简单的 4 步流程，而是一个依赖关系图，必须按拓扑序执行**。

---

## 一、总体计算模型

### 1.1 问题本质

不同干员的效率计算依赖不同级别的全局状态。如果不按正确顺序计算，会出现"用到了还没算出来的值"的逻辑错误。

### 1.2 正确计算流程（5 步，非 4 步）

```
程序开始
  ↓
① 收集所有干员的特殊变量声明（初始化特殊变量字典）
  ↓
② 统计所有跨设施全局计数（宿舍总数、发电站数、贸易站数、派系人数等）
  ↓
③ 第一轮遍历：计算所有特殊技能（产出原始变量值）
  ↓
④ 第二轮遍历：计算全局变量转化（感知信息→思维链环/无声共鸣等）
  ↓
⑤ 第三轮遍历：计算各设施效率（设施基础 + 干员效率）
  ↓
⑥ 第四轮遍历：计算设施产出（制造、贸易、会客室等）
```

**关键发现：步骤②是新发现的隐藏步骤。** 许多干员的特殊技能（如清流"每贸易站+20%"）在计算自身效率前就需要知道其他设施的数量，但这些数量在步骤①中并未被收集。

## 二、干员分类与依赖级别

每个干员根据其效率计算所依赖的"状态来源"，属于以下 7 个依赖级别之一。

### 2.1 依赖级别一览

| 级别 | 名称 | 依赖内容 | 计算时机 | 代表干员 |
|:---:|------|---------|---------|---------|
| **L0** | 无依赖 | 无需任何外部状态 | 任意 | 帕拉斯、酒神 |
| **L1** | 自身精英等级 | 仅根据自身精英等级 | 任意 | 弑君者、能天使 |
| **L2** | 宿舍人数 | 宿舍内干员总数 | 步骤③前需步骤② | 迷迭香、黑键、乌有、塑心 |
| **L3** | 宿舍等级 | 宿舍当前等级（上限5） | 步骤③前需步骤② | 爱丽丝、车尔尼 |
| **L4** | 设施数量 | 发电站数/贸易站数/训练室等级等 | 步骤③前需步骤② | 清流、引星棘刺、森蚺、温蒂、维伊 |
| **L5** | 派系计数 | 某派系在基建中的总人数 | 步骤③前需步骤② | 摩根、杏仁、摆渡人、乌尔比安 |
| **L6** | 归零替代 | 站内其他干员的效率值 | 步骤⑤中动态计算 | 巫恋、冬时、槐琥、雪雉、孑 |

### 2.2 各级别详细说明

#### L0 — 无依赖
固定数值，与任何状态无关。
```
帕拉斯 E2: 生产力 = 30%
酒神 E2: 作战记录效率 = 35%
```

#### L1 — 精英等级依赖
仅根据自身精英等级选择不同固定值。
```
弑君者: E0→赤金+20%, E2→作战记录+35%
能天使: E0→订单+20%, E2→订单+30%
```

#### L2 — 宿舍人数依赖
需要先知道宿舍内有多少干员，才能计算感知信息/人间烟火等变量。
```
迷迭香 E2: 宿舍每1人 → 感知信息+1 → 思维链环/1 = +1%效率
黑键 E2:  宿舍每1人 → 感知信息+1 → 无声共鸣/2 = +0.5%效率
乌有 E2:  宿舍每1人 → 人间烟火+1 → ×1%效率
塑心 E0:  宿舍每1人 → 无声共鸣+1 → /4 = +0.25%效率
```
**注意**：多人宿舍中，感知信息相关干员共享同一份感知信息累加，迷迭香的效率是所有人的贡献总量。

#### L3 — 宿舍等级依赖
需要先知道宿舍的当前等级（设施升级后的等级，1-5）。
```
爱丽丝 E2: 宿舍每1级 → 梦境+1 → 感知信息+1
车尔尼 E2: 宿舍每1级 → 小节+1 → 感知信息+1
```
**注意**：宿舍等级是设施属性，不是干员属性，需要从配置数据中读取。

#### L4 — 设施数量依赖
需要先统计其他设施的数量（发电站数、贸易站数、训练室等级等）。
```
森蚺 E2: 每个发电站 → +10%生产力（归零站内其他人）
温蒂 E2: 每个发电站 → +10%生产力（归零站内其他人）
清流 E1: 每个贸易站 → 贵金属制造站+20%
引星棘刺 E2: 每个贸易站 → 贵金属制造站+3%
维伊 E2:  训练室每级 → 本制造站+10%（上限30%）
伺夜 E2:  会客室每级 → 订单效率+5%（上限40%）
渡桥 E2:  会客室每级 → 订单效率+5%（上限30%）
空弦 E2:  每间宿舍每级 → 订单效率+2%
锡人 E2:  每间宿舍每级 → 联络速度+2%
菲莱 E2:  每间宿舍每级 → 充能速度+0.5%
```
**这是最容易遗漏的依赖类型**。代码中清流/温蒂的 `计算效率()` 目前返回 0.0，就是待外部传入发电站数量的原因。

#### L5 — 派系计数依赖
需要先统计某派系在基建中的总人数。
```
摩根 E2: 每名格拉斯哥帮 → +20%生产力（推进之王额外+35%）
杏仁 E0: 每名黑钢国际 → 贵金属+2%（最多3名）
摆渡人 E2: 每名米诺斯 → 会客室线索速度+5%（最多25%）
歌蕾蒂娅 E2: 每名深海猎人在制造站 → 该制造站+10%（最多90%）
乌尔比安 E2: 每名深海猎人在基建 → 训练速度+10%（最多5名）
```
派系成员列表维护在 `data/skills_by_room.json` 的技能描述文本中。

#### L6 — 站内他人效率依赖
需要先计算出站内其他干员的效率值，才能计算自身效率。这是最高级的依赖。

```
巫恋 E2: 他人效率归零 → 每人+45%效率（贸易站）
冬时 E2: 他人生产力归零 → 每站内干员+10%生产力（制造站）
槐琥 E2: 他人每5%生产力 → 自身额外+5%（上限40%，制造站）
雪雉 E2: 他人每5%效率 → 自身额外+5%（上限35%，贸易站）
孑 E2: 他人每10%效率 → 订单上限-1（贸易站）
```
**处理方式**：此类干员在步骤⑤（计算效率）中，需要先算出站内其他干员的总效率，再代入公式计算自身效率。

## 三、全局计数体系（步骤②）

### 3.1 什么是全局计数

全局计数 = 跨设施的、需要在整个基建范围内统计的数值。

### 3.2 完整全局计数列表

```
宿舍计数:
  - 各宿舍干员总数: dorm_count[宿舍名] = len(进驻干员)
  - 宿舍总干员数:   total_dorm_operators = Σ dorm_count
  - 各宿舍等级:     dorm_level[宿舍名] = 设施等级
  - 宿舍等级总和:   total_dorm_levels = Σ dorm_level

发电站计数:
  - 发电站总数:     power_station_count = len(发电站设施)

贸易站计数:
  - 贸易站总数:     trade_station_count = len(贸易站设施)

会客室计数:
  - 会客室等级:     reception_level = 设施等级

训练室计数:
  - 训练室等级:     training_level = 设施等级

办公室计数:
  - 办公室等级:     office_level = 设施等级

派系计数:
  - 莱茵生命人数:   rheinhart_count = Σ(各派系干员在基建中的总数)
  - 深海猎人人数:   deepsea_count
  - 黑钢国际人数:   blacksteel_count
  - 米诺斯人数:     minos_count
  - 格拉斯哥帮人数: glasgow_count
  - 岁干员人数:     sui_count
  - 杜林族人数:     dolee_count
  - 各派系成员列表维护在 skills_by_room.json 中
```

### 3.3 全局计数计算代码

```python
def 计算全局计数(配置数据: dict) -> dict:
    """步骤②：计算所有跨设施的全局计数"""
    counts = {
        "宿舍总干员数": 0,
        "宿舍等级总和": 0,
        "发电站总数": 0,
        "贸易站总数": 0,
        "会客室等级": 0,
        "训练室等级": 0,
        "办公室等级": 0,
        "各宿舍干员数": {},
        "各宿舍等级": {},
        "派系人数": {},
    }

    for 设施名, 设施 in 配置数据.items():
        类型 = 设施.get("类型")
        等级 = 设施.get("等级", 1)

        if 类型 == "宿舍":
            干员 = 设施.get("进驻干员", [])
            counts["各宿舍干员数"][设施名] = len(干员)
            counts["宿舍总干员数"] += len(干员)
            counts["各宿舍等级"][设施名] = 等级
            counts["宿舍等级总和"] += 等级

        elif 类型 == "发电站":
            counts["发电站总数"] += 1

        elif 类型 == "贸易站":
            counts["贸易站总数"] += 1

        elif 类型 == "会客室":
            counts["会客室等级"] = max(counts["会客室等级"], 等级)

        elif 类型 == "训练室":
            counts["训练室等级"] = max(counts["训练室等级"], 等级)

        elif 类型 == "办公室":
            counts["办公室等级"] = max(counts["办公室等级"], 等级)

    # 派系计数（需遍历所有设施的干员）
    派系 = load_operator_faction_data()
    for 派系名, 成员列表 in 派系.items():
        counts["派系人数"][派系名] = 0
        for 设施名, 设施 in 配置数据.items():
            for op in 设施.get("进驻干员", []):
                if op["名称"] in 成员列表:
                    counts["派系人数"][派系名] += 1

    return counts
```

## 四、全局变量体系

### 4.1 变量分类

全局变量分为两类：

| 类型 | 说明 | 来源 | 消费者 |
|------|------|------|--------|
| **原始产出变量** | 干员技能直接产出 | 步骤③ | 步骤④或⑤ |
| **转化变量** | 由其他变量转化而来 | 步骤④ | 步骤⑤ |

### 4.2 原始产出变量

| 变量名 | 产出条件 | 产出者 | 每点对应效果 |
|--------|---------|--------|------------|
| 感知信息 | 宿舍每1人 | 迷迭香、黑键 | →思维链环→+1%E2 |
| 梦境 | 宿舍每1级 | 爱丽丝 | →感知信息→+1%E2 |
| 小节 | 宿舍每1级 | 车尔尼 | →感知信息→+1%E2 |
| 记忆碎片 | 每额外招募位 | 絮雨 | →感知信息E2 |
| 人间烟火 | 宿舍每1人 | 乌有 | →巫术结晶→+1%E2 |
| 无声共鸣 | 宿舍每1人 | 塑心、深律 | →+0.25%E0效率 |
| 巫术结晶 | 人间烟火每5点 | 截云 | →+1%E1效率 |
| 情报储备 | 每队友 | 灰烬 | →闪击联络+5% |
| 乌萨斯特饮 | 每瓶 | 战车 | →闪击联络+5% |
| 热情值 | 特定干员 | 三角初华等 | →贵金属+1%E2 |
| 魔物料理 | 宿舍等级 | 森西 | →莱欧斯线索+2% |

### 4.3 变量转化链

```
感知信息来源（步骤③）:
  宿舍人数 → 迷迭香(+1/人) + 黑键(+1/人)
  宿舍等级 → 爱丽丝(+1/级) + 车尔尼(+1/级)
  招募位   → 絮雨(+10/位)
  控制中枢 → 夕(心情>12:+10) + 令(心情≤12:+10)

感知信息消耗（步骤④）:
  迷迭香: 感知信息/1 = 思维链环 (E2: 每点+1%)
  黑键:   感知信息/1 = 无声共鸣 (E2: 每2点+1%)

人间烟火来源（步骤③）:
  宿舍人数 → 乌有(+1/人)
  控制中枢 → 夕(心情≤12:+15) + 令(心情>12:+15)
  招募位   → 桑葚(+10/位)
  岁干员   → 重岳(+5/人, 最多5名)

人间烟火消耗（步骤④）:
  截云: (人间烟火 // 5) = 巫术结晶 (E2: 每点+2%)
```

### 4.4 变量来源优先级

当同一变量有多个来源时，**加法叠加**：
```
感知信息 = 迷迭香(4人→+4) + 黑键(4人→+4) + 爱丽丝(5级→+5) + 夕(+10) = 23点
迷迭香E2效率 = 23 × 1% = 23%
```

## 五、各设施计算公式

### 5.1 制造站

```
总生产力(%) = (等级基础 - 1) × 100 + Σ各干员效率

等级基础: Lv.1=1.00, Lv.2=1.00, Lv.3=1.03

效率来源:
  ① 干员固定加成 (L0/L1):     帕拉斯E2+30%、红云E1+15%
  ② 全局变量转化 (L2/L3):     迷迭香(思维链环)、截云(巫术结晶)
  ③ 设施数量联动 (L4):        清流(每贸易站+20%)、维伊(训练室每级+10%)
  ④ 派系计数 (L5):            摩根(格拉斯哥+20%)、杏仁(黑钢+2%×N)
  ⑤ 归零替代 (L6):            冬时(归零+每干员+10%)、槐琥(他人每5%+5%)

归零类干员特殊处理:
  站内存在归零干员时 → 其他干员效率=0 → 设施总效率=归零干员效率

贵金属产线加成:
  控制中枢清流:     每贸易站+20%
  引星棘刺E2:       每贸易站+3%
  制造站娜仁图亚E2: 每宿舍每级+1%
  制造站娜斯提E2:   每莱茵生命+3%(最多5名)
```

#### 5.1.1 制造站完整计算伪代码

```python
def 计算制造站效率(设施, 全局计数, 特殊变量):
    # ① 初始化干员实例
    干员们 = [创建干员实例(op) for op in 设施.进驻干员]

    # ② 检查归零类干员
    归零干员 = [op for op in 干员们 if op.是否归零类()]
    站内其他效率 = Σ([op.基础效率 for op in 干员们 if op not in 归零干员])

    # ③ 计算各干员效率
    效率列表 = []
    for 干员 in 干员们:
        if 干员 in 归零干员:
            # 归零干员：效率 = 自身值 + 设施数量加成 + 站内干员数加成
            效率 = 干员.自身效率
            效率 += 全局计数["发电站总数"] * 干员.每发电站加成
            效率 += len(干员们) * 干员.每站内干员加成
            效率 += 站内其他效率 * 干员.他人放大比例  # 槐琥
        else:
            # 正常干员
            效率 = 干员.基础效率

        效率列表.append(效率)

    总效率 = (STATION_LEVEL_BONUS[设施.等级] - 1) * 100 + Σ(效率列表)
    return 总效率
```

### 5.2 贸易站

```
订单效率(%) = 100 + 干员数×1 + Σ干员效率加成

订单上限 = 等级基础(6/8/10) + Σ上限加成

效率来源:
  ① 干员固定加成 (L0/L1):     能天使E2+30%、德克萨斯+拉普兰德组合+65%
  ② 全局变量转化 (L2):         乌有(人间烟火×1%)
  ③ 设施数量联动 (L4):         伺夜(会客室每级+5%)、空弦(宿舍每级+2%)
  ④ 派系计数 (L5):             摩根(格拉斯哥+20%)、摆渡人(米诺斯+5%×N)
  ⑤ 归零替代 (L6):             巫恋(他人归零+每人45%)、雪雉(他人每5%+5%)

上限加成来源:
  史都华德/暗索/桃金娘:  +5/站
  孑:                   每他人10%效率→-1上限(最低1)
  琳琅诗怀雅E2:         每上限+4%效率
  锏E2:                 每5上限+25%效率(上限100%)
  拉普兰德E2:           德克萨斯同站→+4上限
  灵知E2:               每谢拉格成员→-15%效率+6上限
```

#### 5.2.1 贸易站完整计算伪代码

```python
def 计算贸易站效率(设施, 全局计数, 特殊变量):
    干员们 = [创建干员实例(op) for op in 设施.进驻干员]

    # ① 计算站内其他干员总效率
    站内其他效率 = Σ([op.基础效率 for op in 干员们])

    # ② 计算各干员效率（含归零类特殊处理）
    效率列表 = []
    上限加成 = 0
    for 干员 in 干员们:
        if 干员.是否巫恋类():
            效率 = 45 * (len(干员们) - 1)  # 每人+45%
        elif 干员.是否雪雉类():
            他人效率 = 站内其他效率 - 干员.基础效率
            倍数 = min(他人效率 // 5, 7)  # 最多35%→7倍
            效率 = 干员.基础效率 + 倍数 * 5
        elif 干员.是否孑类():
            他人效率 = 站内其他效率 - 干员.基础效率
            上限减量 = max(1, 他人效率 // 10)
            效率 = 干员.基础效率 + (len(干员们) - 1 - 上限减量) * 4
            上限加成 -= 上限减量
        elif 干员.是否锏类():
            基础上限 = TRADE_STATION_ORDER_LIMIT[设施.等级]
            当前上限 = 基础上限 + 上限加成
            倍数 = min(当前上限 // 5, 4)  # 最多100%
            效率 = 干员.基础效率 + 倍数 * 25
        else:
            效率 = 干员.基础效率

        效率列表.append(效率)

    总效率 = 100 + len(干员们) + Σ(效率列表)
    总上限 = TRADE_STATION_ORDER_LIMIT[设施.等级] + 上限加成
    return {"效率": 总效率, "上限": max(1, 总上限)}
```

## 六、完整计算伪代码

```python
def 运行模拟器(配置数据: dict) -> dict:
    特殊变量 = 特殊变量存储器()

    # ===== 步骤①：收集特殊变量声明 =====
    for 设施 in 所有设施:
        for 干员描述 in 设施.进驻干员:
            干员类 = REGISTRY[干员描述.名称]
            for 变量名 in 干员类.特殊变量列表:
                特殊变量.初始化变量(变量名)

    # ===== 步骤②：计算全局计数 =====
    全局计数 = {
        "宿舍总干员数":   Σ(各宿舍干员数),
        "宿舍等级总和":   Σ(各宿舍等级),
        "发电站总数":     count(发电站),
        "贸易站总数":     count(贸易站),
        "会客室等级":     max(会客室等级),
        "训练室等级":     max(训练室等级),
        "办公室等级":     max(办公室等级),
        "派系人数":       {派系: count(该派系干员) for 派系 in ALL_FACTIONS},
        "各宿舍干员数":   {宿舍名: len(进驻干员)},
        "各宿舍等级":     {宿舍名: 设施等级},
    }

    # ===== 步骤③：计算特殊技能（产出原始变量）=====
    for 设施 in 所有设施:
        for 干员描述 in 设施.进驻干员:
            干员实例 = 创建干员实例(
                干员描述.名称,
                干员描述.精英等级,
                特殊变量,
                配置数据,
                全局计数          # 传入全局计数供 L2/L3/L4/L5 依赖使用
            )
            # 心情依赖变量（夕/令）需先获取当前心情
            干员实例.心情值 = 心情系统.获取(干员描述.名称)
            干员实例.计算特殊技能()

    # ===== 步骤④：转化全局变量 =====
    for 设施 in 所有设施:
        for 干员描述 in 设施.进驻干员:
            干员实例 = 创建干员实例(...)
            if hasattr(干员实例, '计算感知信息转化'):
                干员实例.计算感知信息转化()

    # ===== 步骤⑤：计算各设施效率 =====
    for 设施 in 所有设施:
        设施.特殊变量 = 特殊变量
        设施.全局计数 = 全局计数
        设施.站内干员效率 = {}

        # ⑤-a: 先算所有干员的基础效率（供 L6 归零类使用）
        for 干员描述 in 设施.进驻干员:
            干员实例 = 创建干员实例(...)
            基础效率 = 干员实例.计算效率()
            设施.站内干员效率[干员描述.名称] = 基础效率

        # ⑤-b: 再算归零类干员的最终效率
        for 干员描述 in 设施.进驻干员:
            干员实例 = 创建干员实例(...)
            if 干员实例.是否归零类():
                他人效率 = Σ(设施.站内干员效率.values())
                最终效率 = 干员实例.计算归零效率(他人效率, 全局计数)
                设施.站内干员效率[干员描述.名称] = 最终效率

    # ===== 步骤⑥：计算设施产出 =====
    for 设施 in 所有设施:
        结果[设施名] = 设施.计算产出()

    return 结果
```

### 7.2 关键注意事项

1. **清流/温蒂的效率不为0**：它们返回 0.0 是代码中的占位值。外部调用方（`制造站效率计算器`）需要在步骤⑤中将全局计数中的发电站/贸易站数量传入，计算出真实效率。

2. **槐琥、雪雉、孑 需要两步计算**：第一步算出站内其他干员的基础效率，第二步代入自身公式算出最终效率。两者顺序不可颠倒。

3. **巫恋同时影响心情**：巫恋进驻后，站内所有干员心情消耗+0.25/h。这属于心情系统的副作用，需要在步骤⑥前单独处理。

4. **夕/令 的心情依赖**：默认使用满心情(24)。如需精确模拟，需引入心情动态追踪循环（计算→消耗→更新心情→重新计算）。
